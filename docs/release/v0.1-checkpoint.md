# Runnable v0.1 Checkpoint

**Date:** 2026-07-03
**Decision:** `CONTINUE_FULL_V1`

## Context

Task 14 Step 9 of the initial-release plan requires a formal checkpoint: the
objective-only CLI, installed as a **clean wheel in a fresh environment**, must
run the GSM8K quickstart end-to-end with no importer code, no manual dataset
download, and no `datasets` / `pyarrow` / Docker dependency, produce a canonical
JSON report, and surface provider failures as stable codes without tracebacks.
The checkpoint then records one explicit decision: ship objective-only
(`SHIP_V0_1`) or execute the deferred Slice-4b parts (`CONTINUE_FULL_V1`).

## Decision

**`CONTINUE_FULL_V1`.** The user directed "complete all the work." The Slice-4b
components the plan lists as deferrable — calibrated judges (Task 10 Steps 6-9),
advanced statistics (Task 12), and rich reporters (Task 13 Steps 5-7) — were in
fact implemented in the parallel waves that produced this checkpoint, so only
**Task 14 Steps 10-11** (`compare` / `report` commands) and **Tasks 15-16**
(release gates and acceptance audit) remain.

_Alternative not taken:_ `SHIP_V0_1` — ship objective-only and move judges,
statistics, rich reporters, and comparison commands to a v0.2 plan.

## Verification: clean-wheel quickstart

Built `agentic_evalkit-0.1.0-py3-none-any.whl` with `uv build`, installed it into
a fresh CPython 3.12.11 virtual environment, and confirmed the environment
contains **none** of `datasets`, `pyarrow`, `pytest`, `mypy`, `ruff`, or `torch`.
All commands below were run from a working directory **outside** the repository,
so `agentic_evalkit` resolves to the installed wheel, not the source tree.

```
agentic-evalkit doctor
agentic-evalkit init --preset gsm8k --output eval.yaml
agentic-evalkit run eval.yaml --limit 5 --yes
```

**Run outcomes:** `total=5 passed=0 failed=5 partial=0 errors=0 timeouts=0
cancelled=0 abstained=0 unavailable=0`. Every execution `completed` and every
grade is `fail` — the `zero_target` intentionally returns `"0"`, which is not the

**Canonical report** (`agentic-evalkit-runs/<run_id>.json`) top-level keys:

samples, started_at, finished_at, generated_at`. Provenance carries

No importer code, manual dataset download, `datasets`, `pyarrow`, or Docker was
required at any step.

## Known issues

- `doctor` prints the Hugging Face latency unrounded (e.g. `235.00000000058208ms`). Cosmetic; round in a later polish.
- The plan's `test_provider_failure_has_nonzero_exit_and_error_code` expected `dataset_not_found`. Hugging Face's Dataset Viewer returns HTTP **401** (deliberate anti-enumeration) rather than 404 for a nonexistent dataset, which correctly maps to `dataset_access_denied` (ADR-0003 / Task 6). The exit code (4) is unchanged; the test now asserts exit 4 plus a stable provider error code, with the reason documented in the test module.
- **Resolved 2026-07-05:** provider-facing CLI tests now use canned providers in the default (`not live`) suite, real CLI checks live only under `tests/live/`, and a contract test enforces the split. The end-to-end `run` test no longer resolves GSM8K through the real Dataset Viewer.
